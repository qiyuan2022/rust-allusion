## Phase 3: 文件系统与导入模块

**目标**: 文件夹监控与图片导入

### 3.1 文件监控引擎 (core/monitor.rs)

| 任务 | 描述 | 技术点 |
|------|------|--------|
| 3.1.1 notify 集成 | 跨平台文件监控 | inotify/FSEvents/ReadDirectoryChangesW |
| 3.1.2 事件防抖 | 500ms 聚合批量事件 | 避免风暴 |
| 3.1.3 递归监控 | 支持子文件夹 | 配置最大深度 |
| 3.1.4 监控持久化 | 重启后恢复监控列表 | SQLite 存储 |

### 3.2 图片导入器 (core/importer.rs)

| 任务 | 描述 | 验收标准 |
|------|------|----------|
| 3.2.1 文件扫描 | 遍历文件夹收集图片 | 支持 10万+ 文件 |
| 3.2.2 格式识别 | JPEG/PNG/WebP/RAW/PSD | 可配置格式列表 |
| 3.2.3 文件哈希 | blake3 流式哈希 | 检测重复/变更 |
| 3.2.4 元数据提取 | 图片尺寸、EXIF | 使用 kamadak-exif |
| 3.2.5 批量导入 | 进度报告 + 可取消 | 前端显示进度条 |

### 3.3 Tauri 命令接口

| 命令 | 功能 |
|------|------|
| `add_location(path)` | 添加监控文件夹 |
| `remove_location(id)` | 移除监控 |
| `get_locations()` | 获取监控列表 |
| `import_images(paths)` | 手动导入图片 |
| `get_import_progress()` | 获取导入进度 |

**里程碑**: ✅ 添加文件夹后自动导入图片，数据库记录正确

## Phase 4: 缩略图引擎 (核心性能模块)

**目标**: 快速、低内存的缩略图生成

### 4.1 libvips 集成

| 任务 | 描述 | 注意点 |
|------|------|--------|
| 4.1.1 环境配置 | 系统 libvips 安装脚本 | Windows/macOS/Linux |
| 4.1.2 FFI 绑定 | 使用 rust-bindgen 或现有 crate | 处理 unsafe 代码 |
| 4.1.3 错误处理 | C 错误码转 Rust Result | 内存安全 |

### 4.2 缩略图引擎 (core/thumbnail.rs)

| 任务 | 描述 | 验收标准 |
|------|------|----------|
| 4.2.1 缓存目录管理 | 分级存储结构 | 自动清理过期缓存 |
| 4.2.2 并发控制 | Semaphore 限制并发数 | CPU核心数 * 2 |
| 4.2.3 多尺寸生成 | small(200px)/medium(800px)/large(1600px) | 按需生成 |
| 4.2.4 流式处理 | 不加载全图到内存 | 100MB 图片内存 < 20MB |
| 4.2.5 缓存索引 | 数据库记录缩略图状态 | 避免重复生成 |
| 4.2.6 后台队列 | 优先处理可见区域 | 滚动时优先生成 |

### 4.3 自定义 Asset 协议

```rust
// 支持 URL: asset://thumbnail/{image_id}/{size}
// 前端直接通过 <img src="asset://..." /> 显示
```

**里程碑**: ✅ 滚动 1000 张图片，缩略图生成不卡顿，内存稳定

## MVP 范围定义

如需快速验证方案，可优先实现 **MVP 核心功能**：

```
✅ Phase 1 (基础)
✅ Phase 2 (数据层)
✅ Phase 3 (文件监控 + 导入)
✅ Phase 4 (缩略图 - 可用版本)
    - 简化：单尺寸缩略图
✅ Phase 5 (标签 - 基础版)
    - 简化：单层标签，无层级
✅ Phase 7 (前端 - 画廊 + 预览)
    - 简化：基础虚拟滚动，无高级搜索
```

**MVP 功能清单**:
- 添加监控文件夹
- 自动导入图片（基础格式）
- 单尺寸缩略图生成与显示
- 基础虚拟滚动画廊
- 单层标签系统
- 图片预览
- 基础搜索（文件名）
